# TodoListApp 도메인 정의서

**버전:** 1.0.0
**작성일:** 2026-05-12
**상태:** 초안

---

## 1. 프로젝트 개요

### 1.1 문제 정의

현재 시장에는 다음과 같은 문제를 가진 할일 관리 환경이 존재한다.

- 개인 단위로 할일을 독립적으로 관리할 수 있는 적절한 도구가 부재하다.
- 인증된 사용자 기반으로 데이터베이스에 할일을 저장하고 보호하는 앱이 없다.
- 할일을 카테고리별로 분류하여 등록하고 조회할 수 있는 구조적 관리 수단이 없다.

### 1.2 목적

TodoListApp은 사용자 인증 기반의 개인 할일 관리 플랫폼이다. 각 사용자는 자신만의 할일 목록을 안전하게 생성, 분류, 추적할 수 있다. 카테고리와 다양한 필터 조건을 통해 할일을 체계적으로 관리함으로써 개인 생산성 향상을 목표로 한다.

## 2. 도메인 모델

### 2.1 핵심 엔티티

#### User (사용자)

인증 및 개인 데이터 소유의 주체. 모든 할일 데이터는 사용자에게 귀속된다.

| 속성 | 타입 | 설명 |
|---|---|---|
| userId | UUID | 사용자 고유 식별자 |
| email | String | 로그인 ID로 사용되는 이메일 주소 (유일값) |
| password | String | 암호화된 비밀번호 |
| name | String | 사용자 이름 |
| createdAt | DateTime | 계정 생성 일시 |
| updatedAt | DateTime | 계정 정보 최종 수정 일시 |

---

#### Category (카테고리)

할일을 분류하는 기준 단위. 시스템이 제공하는 기본 카테고리와 사용자가 직접 생성하는 사용자 정의 카테고리로 구분된다.

| 속성 | 타입 | 설명 |
|---|---|---|
| categoryId | UUID | 카테고리 고유 식별자 |
| userId | UUID | 소유 사용자 ID (기본 카테고리의 경우 null) |
| name | String | 카테고리 명칭 |
| isDefault | Boolean | 시스템 기본 카테고리 여부 |
| createdAt | DateTime | 카테고리 생성 일시 |

---

#### Todo (할일)

사용자가 등록하는 할일의 핵심 단위. 제목, 설명, 종료 예정일, 카테고리, 완료 여부를 포함한다.

| 속성 | 타입 | 설명 |
|---|---|---|
| todoId | UUID | 할일 고유 식별자 |
| userId | UUID | 소유 사용자 ID |
| categoryId | UUID | 분류 카테고리 ID |
| title | String | 할일 제목 |
| description | String | 할일 상세 설명 (선택) |
| dueDate | Date | 종료 예정일 |
| isCompleted | Boolean | 완료 여부 |
| completedAt | DateTime | 실제 완료 일시 (완료 시 기록) |
| createdAt | DateTime | 할일 생성 일시 |
| updatedAt | DateTime | 할일 최종 수정 일시 |

---

### 2.2 엔티티 관계 요약

```
User (1) ─────────────── (N) Todo
User (1) ─────────────── (N) Category (사용자 정의)
Category (1) ─────────── (N) Todo
System ────────────────── (N) Category (기본 카테고리, 전체 사용자 공유)
```

## 3. 핵심 비즈니스 규칙

### 3.1 사용자 및 인증

- **BR-01** 이메일 주소는 시스템 전체에서 유일해야 한다. 중복 이메일로는 회원가입이 불가하다.
- **BR-02** 비밀번호는 반드시 암호화하여 저장한다. 평문 저장은 허용되지 않는다.
- **BR-03** 인증되지 않은 사용자는 할일 및 카테고리에 접근할 수 없다.
- **BR-04** 사용자는 자신의 계정 정보(이름, 비밀번호)만 수정할 수 있다. 타인의 정보에는 접근 불가하다.

### 3.2 카테고리

- **BR-05** 기본 카테고리는 시스템이 제공하며, 모든 사용자에게 공통으로 제공된다. 사용자가 삭제하거나 수정할 수 없다.
- **BR-06** 사용자는 기본 카테고리 외에 자신만의 카테고리를 추가로 생성할 수 있다.
- **BR-07** 사용자 정의 카테고리는 해당 사용자만 조회, 수정, 삭제할 수 있다.
- **BR-08** 카테고리를 삭제할 경우, 해당 카테고리에 속한 할일의 카테고리는 기본 카테고리로 자동 재분류된다.

### 3.3 할일 (Todo)

- **BR-09** 할일은 반드시 제목과 카테고리를 포함해야 한다. 설명과 종료 예정일은 선택 항목이다.
- **BR-10** 할일은 반드시 인증된 사용자에게 귀속되며, 소유자만 조회, 수정, 삭제할 수 있다.
- **BR-11** 할일을 완료 처리하면 완료 일시(completedAt)가 기록된다. 완료 취소 시 해당 값은 초기화된다.
- **BR-12** 종료 예정일이 오늘 날짜를 초과한 미완료 할일은 "기한 초과" 상태로 간주된다.

### 3.4 할일 목록 조회 필터

- **BR-13** 할일 목록은 다음 조건을 단독 또는 조합하여 필터링할 수 있다.
  - 카테고리 기준 필터
  - 종료 예정일 기간 기준 필터 (시작일 ~ 종료일 범위)
  - 완료 여부 기준 필터 (완료 / 미완료 / 전체)

| UC-02 | 로그인 | 비인증 사용자 | 이메일과 비밀번호로 인증 토큰을 발급받는다. |

## 5. 용어 정의 (Ubiquitous Language)

| 용어 | 영문 | 정의 |
|---|---|---|
| 사용자 | User | 회원가입 후 인증을 통해 앱을 사용하는 주체. 모든 데이터의 소유 단위이다. |
| 인증 | Authentication | 이메일과 비밀번호를 통해 사용자의 신원을 확인하는 행위. |
| 할일 | Todo | 사용자가 완료를 목표로 등록하는 작업 단위. 제목, 설명, 종료 예정일, 카테고리로 구성된다. |
| 기본 카테고리 | Default Category | 시스템이 모든 사용자에게 공통 제공하는 카테고리. 사용자가 수정하거나 삭제할 수 없다. |
| 사용자 정의 카테고리 | Custom Category | 사용자가 직접 생성한 개인 카테고리. 해당 사용자만 관리할 수 있다. |
| 종료 예정일 | Due Date | 할일을 완료하기로 계획한 목표 날짜. |
| 완료 여부 | Completion Status | 할일이 완료되었는지를 나타내는 상태값. 완료(true) 또는 미완료(false)로 구분된다. |
| 기한 초과 | Overdue | 종료 예정일이 현재 날짜를 초과한 미완료 상태의 할일. |
| 필터 | Filter | 할일 목록 조회 시 카테고리, 기간, 완료 여부 기준으로 결과를 좁히는 조건. |
