## 1. 화면 목록 및 네비게이션 흐름

### 1.1 화면 목록

| 화면 ID | 화면명             | 경로          | 인증      | 설명                           |
| ------- | ------------------ | ------------- | --------- | ------------------------------ |
| P-01    | 로그인 화면        | `/login`      | 비인증    | 이메일/비밀번호 입력 폼        |
| P-02    | 회원가입 화면      | `/signup`     | 비인증    | 이름/이메일/비밀번호 입력 폼   |
| P-03    | 할일 목록 화면     | `/todos`      | 인증 필요 | 메인 화면, 필터바, 할일 리스트 |
| P-04    | 카테고리 관리 화면 | `/categories` | 인증 필요 | 카테고리 추가/관리             |
| P-05    | 마이페이지 화면    | `/profile`    | 인증 필요 | 개인정보 수정                  |
| P-06    | 404 화면           | `*`           | 공개      | 잘못된 경로                    |

### 1.2 네비게이션 흐름

```
┌─────────────────────────────────────────────────────────┐
│              TodoListApp 네비게이션 흐름                   │
├─────────────────────────────────────────────────────────┤
│                                                           │
│  [시작] → /login  (P-01 로그인 화면)                      │
│            ↓                                             │
│         ┌─ "회원가입" 링크→ /signup (P-02 회원가입)     │
│         │   ↓                                             │
│         │ [가입 완료] → /login (자동 리다이렉트)        │
│         │                                                │
│         └─ [로그인 성공] → /todos (P-03 메인)           │
│             ↓                                             │
│          [헤더 네비게이션]                               │
│             ├─ "할일 목록" → /todos                     │
│             ├─ "카테고리" → /categories (P-04)          │
│             ├─ "마이페이지" → /profile (P-05)           │
│             └─ "로그아웃" → /login                      │
│             ↓                                             │
│          [잘못된 경로] → /404 (P-06)                    │
│             ↓                                             │
│          [페이지 새로고침] → /login (토큰 초기화)       │
│                                                           │
└─────────────────────────────────────────────────────────┘
```

### 3.3 상태별 UX

#### 정상 상태 (기본)

- 입력 필드: 테두리 회색, placeholder 텍스트 표시
- 로그인 버튼: 파란색, 활성화 상태

#### 입력 중 (포커스)

- 포커스된 필드: 테두리 파란색, 배경 약간 밝음

#### 검증 오류

```
┌────────────────────────┐
│ 이메일                  │
│ [________________]      │
│ ⚠ 이메일 형식이        │  ← 빨간색 에러 메시지
│   올바르지 않습니다.    │
│                        │
│ 비밀번호                │
│ [________________]      │
│ ⚠ 비밀번호는 필수      │  ← 빨간색 에러 메시지
│   입력 항목입니다.      │
└────────────────────────┘
```

#### 로딩 상태

- 로그인 버튼: "로그인" → "로그인 중..." 텍스트 변경 + 로딩 스피너

#### 오류 상태

```
┌─────────────────────────────┐
│ ⚠ 이메일 또는 비밀번호가     │  ← 위쪽에 붉은색 에러 토스트
│   올바르지 않습니다.         │
└─────────────────────────────┘
[입력 폼 표시...]
```

## 14. 애니메이션 & 인터랙션

### 14.1 페이드/트랜지션

```
모달 열기/닫기:
─ 배경 오버레이: 투명 → 반투명 (200ms ease-out)
─ 모달 박스: opacity 0 → 1 + scale 0.95 → 1 (250ms ease-out)

토스트 메시지:
─ 나타나기: slide-up + fade-in (300ms ease-out)
─ 사라지기: slide-down + fade-out (300ms ease-out, 2초 자동 닫기)

필터 바:
─ 펼침/닫힘: max-height 0 → auto (200ms ease-out)
```

### 14.2 상호작용 피드백

```
버튼:
─ 기본: 배경색 + 그림자
─ 호버: 배경 어두워짐 + 커서 pointer
─ 클릭: 약간의 scale 감소 (98%) + 짧은 transition

체크박스:
─ 클릭 시: 즉시 토글 + 애니메이션 스케일 (90% → 100%)

입력 필드:
─ 포커스: 테두리 색상 변경 + 아래쪽 언더라인 표시 (파란색)
─ 오류: 테두리 빨간색 + 배경 연한 빨강

필터 적용:
─ 할일 리스트 새로고침 시: fade-out → fade-in (200ms)
```
